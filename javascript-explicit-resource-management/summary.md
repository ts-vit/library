# Выжимка: Explicit Resource Management in JavaScript (2026 Update)

Статья описывает внедрение операторов `using` и `await using` в стандарт JavaScript (ECMAScript), что кардинально упрощает управление жизненным циклом ресурсов.

### Ключевые моменты:
1.  **Автоматическая очистка:** Теперь не нужно писать `try / finally` для закрытия файлов или соединений. Ресурс, объявленный через `using`, очищается автоматически при выходе из области видимости.
2.  **Символы `Symbol.dispose` и `Symbol.asyncDispose`:** Это новые стандарты интерфейсов для объектов, которые требуют очистки. Любой класс, реализовавший эти методы, может быть использован с `using`.
3.  **Порядок стека:** Если объявлено несколько ресурсов, они закрываются в обратном порядке (Last-In, First-Out).
4.  **DisposableStack:** Специальный класс для управления группой ресурсов, когда их нельзя привязать к простому блоку кода.

### Применимость в наших проектах:
-   **toshik-babe (Backend):** Идеально для управления соединениями с SQLite и пулами запросов к моделям. Позволит избежать утечек при ошибках в асинхронных цепочках.
-   **toshik-chat (Frontend):** Можно использовать для автоматической отписки от WebSocket-событий или очистки Observers при размонтировании компонентов (хотя в React есть свои хуки, это полезно на уровне чистой логики).

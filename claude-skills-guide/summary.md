# Выжимка: The Complete Guide to Building Skills for Claude (Anthropic)

Это официальное руководство по созданию "скиллов" (Skills) — структурированных наборов инструкций и ресурсов, которые обучают Claude новым рабочим процессам.

### Основные концепции:
1.  **Что такое Skill:** Это папка, содержащая файл `SKILL.md` (обязательно), а также опциональные папки `scripts/`, `references/` и `assets/`.
2.  **Прогрессивное раскрытие (Progressive Disclosure):** Трехуровневая система загрузки для экономии токенов:
    *   **Level 1 (YAML Frontmatter):** Метаданные, которые Claude видит всегда, чтобы понять, когда активировать скилл.
    *   **Level 2 (SKILL.md body):** Основные инструкции, загружаются только при активации.
    *   **Level 3 (Linked files):** Дополнительные справочники и скрипты, которые Claude читает по мере необходимости.
3.  **Связь с MCP (Model Context Protocol):**
    *   **MCP** — это "кухня" (инструменты, доступ к данным).
    *   **Skills** — это "рецепты" (как именно использовать эти инструменты для достижения результата).

### Категории скиллов:
1.  **Создание контента (Asset Creation):** Гайдлайны по стилю, шаблоны документов (frontend-дизайн, генерация отчетов).
2.  **Автоматизация воркфлоу (Workflow Automation):** Пошаговые процессы с проверками (например, `skill-creator`).
3.  **Улучшение MCP (MCP Enhancement):** Обучение Claude эффективной работе с конкретными MCP-серверами (например, Sentry или Linear).

### Технические требования:
*   Файл должен называться строго `SKILL.md` (регистр важен).
*   Имя папки и имя в YAML должны быть в `kebab-case`.
*   Описание (description) во фронтматтере — критический элемент, определяющий триггеры активации.

### Для кого это руководство:
Для разработчиков и опытных пользователей, которые хотят стандартизировать работу Claude, создавая переносимые и надежные автоматизации, работающие одинаково в Claude.ai, Claude Code и через API.

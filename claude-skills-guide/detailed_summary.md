# Claude Skills: Полное руководство (Перевод и адаптация)

Это руководство объясняет, как создавать "навыки" (skills) для Claude — наборы инструкций и инструментов, упакованные в одну папку.

## 1. Основы (Fundamentals)

### Что такое навык?
Это папка, содержащая:
*   `SKILL.md` (обязательно): Основной файл с инструкциями и метаданными (YAML).
*   `scripts/` (опционально): Скрипты на Python, Bash и др.
*   `references/` (опционально): Документация, которую Claude загружает по мере необходимости.
*   `assets/` (опционально): Шаблоны, изображения, шрифты.

### Три уровня раскрытия (Progressive Disclosure)
1.  **YAML Frontmatter**: Всегда в системном промпте. Позволяет Claude понять, *когда* использовать навык.
2.  **SKILL.md Body**: Загружается, когда Claude решает использовать навык. Содержит полные инструкции.
3.  **Linked Files**: Дополнительные файлы, к которым Claude обращается только при необходимости.

## 2. Планирование и дизайн

### Категории навыков
*   **Создание контента**: Документы, презентации, код по стандартам.
*   **Автоматизация процессов**: Многошаговые рабочие процессы.
*   **Улучшение MCP**: Инструкции по эффективному использованию инструментов конкретного MCP-сервера.

### Технические требования
*   Имя папки и поле `name`: `kebab-case` (только строчные, через дефис).
*   Файл: Только `SKILL.md` (регистр важен).
*   **Никакого README.md внутри папки навыка!**

## 3. Написание SKILL.md

### Описание (Description) — это критично!
Должно содержать: **Что делает** + **Когда использовать** (триггерные фразы).
*Пример:* `Manages Linear project workflows. Use when user mentions "sprint", "Linear tasks", or asks to "create tickets".`

### Структура инструкций
1.  **Алгоритм шагов**: Четкая последовательность действий.
2.  **Примеры**: "Пользователь говорит X -> Твои действия Y".
3.  **Обработка ошибок**: Что делать, если MCP не отвечает или валидация не прошла.

## 4. Паттерны проектирования

1.  **Sequential Orchestration**: Жесткая последовательность шагов (Шаг 1 -> Шаг 2).
2.  **Multi-MCP Coordination**: Координация между разными сервисами (взял из Figma -> залил в Drive -> создал задачу в Jira).
3.  **Iterative Refinement**: Цикл "Черновик -> Проверка скриптом -> Правка" до достижения качества.
4.  **Domain Intelligence**: Добавление специфических знаний (например, правил финансового комплаенса) поверх обычного доступа к данным.

## 5. Распространение и установка

*   **Локально**: Положить в папку `skills` вашего OpenClaw или Claude Code.
*   **Claude.ai**: Загрузить ZIP-архив в настройках (Settings > Capabilities > Skills).
*   **Для команд**: Админы могут деплоить навыки на весь воркспейс.

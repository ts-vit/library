# Детальная выжимка по главам: Введение в AutoML

Академическое руководство по созданию систем, которые автоматизируют процесс машинного обучения через метаобучение и внешнюю память.

---

### Глава 1. Проблема автоматизации ML
*   **Суть:** Машинное обучение требует огромных человеческих усилий по подбору архитектур и параметров. AutoML стремится убрать "человека из цикла".
*   **Метаобучение:** Основная концепция — система должна накапливать опыт решения задач, чтобы каждая следующая задача решалась быстрее и точнее.

### Глава 2. Основы обучения и адаптации
*   **Традиционное обучение:** Оптимизация под одну задачу с нуля.
*   **Few-shot Learning:** Способность модели выучить новую концепцию по 1-5 примерам (как это делают люди и современные LLM).
*   **Индуктивное смещение:** Использование знаний о структуре мира для упрощения обучения.

### Глава 3. Нейронные сети с внешней памятью (MANN)
*   **Memory-Augmented Neural Networks:** Попытка объединить гибкость нейросетей с надежностью классических алгоритмов через внешние хранилища данных.
*   **Нейронные машины Тьюринга (NTM):** Архитектуры, которые могут читать и записывать данные в память, подобно компьютеру.

### Глава 4. Алгоритм MAML (Model-Agnostic Meta-Learning)
*   **Концепция:** Мы ищем не "лучшие веса" для задачи, а "лучшую стартовую точку", из которой можно быстро прийти к решению любой задачи за 1-2 шага градиентного спуска.
*   **Применение:** Идеально подходит для систем, которым нужно мгновенно подстраиваться под нового пользователя.

### Глава 5. Метрическое обучение (Matching & Prototypical Networks)
*   **Matching Networks:** Использование механизмов внимания для сравнения новых данных с примерами в памяти.
*   **Прототипы:** Создание "усредненного образа" класса (например, "идеальная собака") и сравнение новых объектов с этим эталоном.

### Глава 6. Автоматизация выбора гиперпараметров
*   **Байесовская оптимизация:** Умный поиск лучших настроек модели без перебора всех вариантов.
*   **NAS (Neural Architecture Search):** Когда один ИИ проектирует архитектуру другого ИИ.

### Глава 7. Активное обучение (Active Learning)
*   **Механика:** Модель сама выбирает, какие данные ей нужно разметить человеку, чтобы учиться эффективнее всего. Она спрашивает только о том, в чем меньше всего уверена.

# Детальная выжимка по главам: Разработка с ИИ (Нэйтан Крокер)

Практическое руководство (2025) о том, как превратить LLM в полноценного партнера по парному программированию.

---

### Глава 1. ИИ как напарник по программированию
*   **Смена роли:** От простого гугления к диалогу. Почему важно относиться к ИИ как к джуниор-разработчику с бесконечной памятью.
*   **Инструментарий:** Сравнение ChatGPT, GitHub Copilot и Cursor. Как выбрать инструмент под задачу.
*   **Ограничения:** Понимание галлюцинаций в коде и важность ручной проверки (Code Review).

### Глава 2. Проектирование системы и архитектура
*   **От идеи к схеме:** Использование ИИ для создания архитектурных диаграмм (Mermaid) и описания ITAM-систем.
*   **Моделирование предметной области (DDD):** Как заставить ИИ помочь с выделением сущностей и агрегатов.
*   **Документирование:** Автоматическая генерация ADR (Architecture Decision Records).

### Глава 3. Написание чистого кода и паттерны
*   **Реализация паттернов:** Применение Factory, Builder и Singleton с помощью подсказок ИИ.
*   **Гексагональная архитектура:** Как структурировать проект на TypeScript/Node.js, чтобы логика была отделена от внешних API (идеально для `toshik-babe`).
*   **Рефакторинг:** Использование ИИ для упрощения сложных функций и удаления "дурно пахнущего кода" (code smells).

### Глава 4. Тестирование с помощью ИИ
*   **Юнит-тесты:** Генерация тестовых наборов для Vitest/Jest. Почему ИИ лучше справляется с краевыми случаями (edge cases).
*   **Интеграционные тесты:** Мокание внешних зависимостей (баз данных, API) с помощью промптов.
*   **TDD с ИИ:** Сначала пишем тест вместе с моделью, затем просим её написать реализацию.

### Глава 5. Инфраструктура и DevOps
*   **Docker:** Написание и оптимизация Dockerfile для контейнеризации приложений.
*   **Terraform:** Описание инфраструктуры как кода (IaC) для деплоя в облака (AWS/Azure).
*   **CI/CD:** Создание GitHub Actions пайплайнов для автоматической сборки и тестирования.

### Глава 6. Безопасность и анализ кода
*   **Моделирование угроз:** Просьба к ИИ "взломать" предложенную архитектуру и найти слабые места.
*   **Поиск уязвимостей:** Статический анализ кода на предмет SQL-инъекций и утечек памяти.
*   **Объяснение чужого кода:** Быстрое погружение в легаси-проекты через суммаризацию функций.

### Глава 7. Локальный запуск и приватность
*   **Зачем это нужно:** Работа с чувствительными корпоративными данными без отправки в облако.
*   **Инструменты:** Настройка Ollama, Llama.cpp и GPT-4All.
*   **Производительность:** Сравнение скорости работы локальных моделей против API.

### Глава 8. Будущее разработки
*   **Программирование на естественном языке:** К чему мы идем и останутся ли программисты?
*   **Заключение:** ИИ не заменит разработчиков, но разработчики, использующие ИИ, заменят тех, кто этого не делает.

---
**Связь с проектом Toshik:** Главы про гексагональную архитектуру и автоматизацию тестов — это то, что мы внедрим в `toshik-babe` следующим шагом для повышения его стабильности.
